## Overview

APOCODECK is a self-contained, offline-first cyberdeck built on a **Raspberry Pi 5 8GB** (while the Raspberry Pi 5 is recommended, it is not necessary) running **Debian Bookworm 64-bit (ARM64)** (The chosen OS is necessary so as not to cause conflicts with the packages). It is designed to operate fully independently of the internet ‚ÄĒ combining offensive cybersecurity tooling, offline reference systems, emergency communications, and survival utilities into a single portable platform.

Whether the scenario is a penetration test in a dead zone, a grid-down emergency, or a field operation with no infrastructure, APOCODECK is built to function.

## Setup

The entire stack is deployed via a single modular bash script:

```bash
sudo bash cyberdeck_setup.sh
```

The script will prompt you to select which modules to install. All modules are optional and independent.

### Prerequisites

- Fresh Raspberry Pi OS Bookworm 64-bit install
- Internet connection for initial package download
- Run as root (`sudo`)

## Modules

### Module 1 ‚ÄĒ Offensive Cyber

A full offensive security toolkit for network reconnaissance, exploitation, wireless attacks, and password cracking.

**Network Scanning**
- `nmap` ‚ÄĒ port scanning, service detection, OS fingerprinting
- `masscan` ‚ÄĒ high-speed large-range port scanner
- `netcat` ‚ÄĒ raw TCP/UDP connections, reverse shells, file transfer
- `socat` ‚ÄĒ advanced relay and tunneling
- `proxychains4` ‚ÄĒ route traffic through proxy chains or Tor
- `tcpdump` ‚ÄĒ command-line packet capture
- `wireshark` / `tshark` ‚ÄĒ deep packet inspection
- `arp-scan` ‚ÄĒ local network host discovery
- `nbtscan` ‚ÄĒ NetBIOS/Windows host discovery

**Web Application Testing**
- `nikto` ‚ÄĒ automated web server vulnerability scanner
- `sqlmap` ‚ÄĒ automated SQL injection and database extraction
- `dirb` / `gobuster` ‚ÄĒ hidden directory and file brute-forcing
- `whatweb` ‚ÄĒ web technology fingerprinting
- `ffuf` ‚ÄĒ fast web fuzzer

**Password & Cracking**
- `john` (John the Ripper) ‚ÄĒ multi-format offline password cracker
- `hydra` ‚ÄĒ network login brute-forcer (SSH, FTP, HTTP, RDP, and more)
- `medusa` ‚ÄĒ parallel network brute-forcer
- `hashcat` ‚ÄĒ GPU/CPU accelerated hash cracking
- **SecLists** ‚ÄĒ comprehensive wordlist collection at `/usr/share/seclists/`
- **rockyou.txt** ‚ÄĒ classic 14M password wordlist at `/usr/share/wordlists/rockyou.txt`

**Wireless**
- `aircrack-ng` ‚ÄĒ WEP/WPA capture and cracking suite
- `reaver` / `bully` ‚ÄĒ WPS PIN brute-force
- `macchanger` ‚ÄĒ MAC address spoofing
- `bettercap` ‚ÄĒ MitM framework, ARP/DNS spoofing, credential capture

**Exploitation**
- `metasploit-framework` ‚ÄĒ industry-standard exploitation framework
- `bettercap` ‚ÄĒ active network attack framework

**Recon**
- `fierce` / `dnsrecon` ‚ÄĒ DNS enumeration and subdomain discovery
- `whois` ‚ÄĒ domain and IP registration lookup
- `traceroute` ‚ÄĒ network path mapping

### Module 2 ‚ÄĒ Offline Maps

Serve interactive OpenStreetMap tiles locally with no internet required.

- **tileserver-gl-light** ‚ÄĒ serves `.mbtiles` files as interactive map tiles over HTTP
- **QGIS** ‚ÄĒ full desktop GIS application for geographic data analysis
- **QMapShack** ‚ÄĒ lightweight map viewer for hiking/navigation maps

**Map tile server** runs at `http://192.168.50.1:8080` when the hotspot is active.

**To add maps:** Download `.mbtiles` files for your region and place them in `/opt/maps/mbtiles/`
- OpenMapTiles: https://openmaptiles.com/downloads/
- Geofabrik: https://download.geofabrik.de/

**Start map server:**
```bash
systemctl start mapserver
```

---

### Module 3 ‚ÄĒ Kiwix + WiFi Hotspot

Serve offline reference content (Wikipedia, medical guides, survival manuals, Stack Overflow, etc.) over a local WiFi hotspot to any connected device.

**Components:**
- `kiwix-serve` ‚ÄĒ serves `.zim` files as a local website
- `hostapd` ‚ÄĒ turns the Pi's WiFi into an access point
- `dnsmasq` ‚ÄĒ provides DHCP and DNS for hotspot clients

**Hotspot credentials** are set during installation (prompted at setup time).

**Quick commands:**
```bash
hotspot-start    # bring up hotspot + Kiwix + maps
hotspot-stop     # shut down hotspot
```

**Kiwix library** available at `http://192.168.50.1:8888` for all connected clients.

**To add content:** Place `.zim` files in `/opt/kiwix/zims/`
- Download ZIMs: https://download.kiwix.org/zim/
- Recommended: Wikipedia, WikiMed (medical), Wikivoyage, Stack Overflow

---

### Module 4 ‚ÄĒ Comms / SDR / Radio

Software-defined radio reception, digital radio modes, and off-grid mesh communications.

- **rtl-sdr** ‚ÄĒ driver for RTL-SDR USB dongles (~$25), enabling wideband radio reception
- **GNU Radio** ‚ÄĒ signal processing framework for building custom radio pipelines
- **GQRX** ‚ÄĒ GUI SDR receiver; tune FM, aviation, weather satellites, and more
- **dump1090** ‚ÄĒ ADS-B decoder; tracks nearby aircraft with no internet
- **fldigi** ‚ÄĒ digital radio modes (PSK31, RTTY, Olivia) ‚ÄĒ pass text over audio
- **Direwolf** ‚ÄĒ APRS software modem for packet radio over audio
- **CHIRP** ‚ÄĒ program Baofeng and other handheld radios from the Pi
- **Meshtastic CLI** ‚ÄĒ manage LoRa mesh radio modules for off-grid encrypted messaging
- **ax25-tools** ‚ÄĒ AX.25 packet radio stack

**Recommended hardware:** RTL-SDR v3 dongle (~$25), Baofeng UV-5R handheld, Meshtastic LoRa module

---

### Module 5 ‚ÄĒ Survival Utilities

General-purpose offline tools for long-term field operation.

- **KeePassXC** ‚ÄĒ encrypted offline password vault
- **Syncthing** ‚ÄĒ peer-to-peer file sync across devices with no server
- **Ollama** ‚ÄĒ run large language models locally; pull `llama3.2:3b` for an offline AI assistant
- **ffmpeg** ‚ÄĒ video/audio conversion and processing
- **VLC** / **mpv** ‚ÄĒ media playback
- **Calibre** ‚ÄĒ offline ebook library management
- **LibreOffice** ‚ÄĒ full offline office suite
- **Stellarium** ‚ÄĒ offline planetarium; identify stars and planets in the field
- **gpsd** ‚ÄĒ GPS daemon for USB GPS receivers
- **OpenVPN** / **WireGuard** ‚ÄĒ VPN and private network tunneling
- **Rsync** ‚ÄĒ efficient file backup and sync

**Pull an offline AI model after install:**
```bash
ollama pull llama3.2:3b
ollama run llama3.2:3b
```

---

### Module 6 ‚ÄĒ System Hardening

- **UFW** ‚ÄĒ firewall; blocks all inbound except SSH (22), Kiwix (8888), Maps (8080)
- **fail2ban** ‚ÄĒ auto-bans IPs with repeated failed login attempts
- **SSH hardening** ‚ÄĒ root login disabled, max auth tries limited
- **hostapd.conf** ‚ÄĒ locked to root-only read (`chmod 600`)
- **Unattended-upgrades** ‚ÄĒ automatic security patch application
- **LUKS** (recommended) ‚ÄĒ full disk encryption; set up during OS install

## Directory Structure

```
/usr/share/seclists/          # SecLists wordlists
/usr/share/wordlists/         # rockyou.txt symlink
/opt/kiwix/zims/              # ZIM files for Kiwix
/opt/maps/mbtiles/            # MBTiles for offline maps
/etc/hostapd/hostapd.conf     # Hotspot config (root-only)
/etc/dnsmasq.d/hotspot.conf   # DHCP/DNS config
/usr/local/bin/hotspot-start  # Hotspot start script
/usr/local/bin/hotspot-stop   # Hotspot stop script
/var/log/cyberdeck_setup.log  # Full install log
```

- [ ] Change system user password: `passwd`
- [ ] Download `.zim` files to `/opt/kiwix/zims/`
- [ ] Download `.mbtiles` for your region to `/opt/maps/mbtiles/`
- [ ] Pull Ollama model: `ollama pull llama3.2:3b`

## Security Notes

- This device is designed for authorized penetration testing and personal resilience use only
- All offensive tools should only be used on networks and systems you own or have explicit written permission to test
- The hotspot password is stored in plaintext at `/etc/hostapd/hostapd.conf` (root-only read) ‚ÄĒ this is a limitation of hostapd, not a misconfiguration
- Full disk encryption (LUKS) is strongly recommended and should be configured during the initial OS flash
